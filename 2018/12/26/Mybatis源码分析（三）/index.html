动态sql标签—-for each
for each标签的主要参数：item index collection open seperator close
item：表示集合中示每一个元素进行迭代时的别名
index：表示集合中元素进行迭代时的位置
collection：表示传入参数的类型
open/close：表示以什么开始以什么结束
sepeator：表示每次迭代之间以什么作为分割符

Mybatis的级联操作：一般来说只会使用级联查询，其他的如级联删除等维护的成本较大，一般不会使用
级联查询—一对多查询（部门和职员之间的一对多的关系 在部门的实体类中声明list类型的数组，里面是职员类型的对象，有两种方式）

级联查询–多对一查询（根据职员的编号查询员工的基本信息和部门的基本信息，员工的实体类型中有一个部门类型的参数）

Mybatis分页（PageHelper插件）

1.引入pagehelper的依赖
< dependency >
    < groupId>com.github.pagehelper< /groupId >
    < artifactId >pagehelper< /artifactId >
    < version >4.1.6< /version >
< /dependency >
2.在Mybatis配置文件中进行配置（plugins应该在environments的上面）
< plugin interceptor=”com.github.pagehelper.PageHelper” >
    < property name=”dialect” value=”mysql”/ >
    < property name=”offsetAsPageNum” value=”false”/ >
    < property name=”rowBoundsWithCount” value=”false”/ >
    < property name=”pageSizeZero” value=”true”/>
    < property name=”reasonable” value=”false”/ >
    < property name=”supportMethodsArguments” value=”false”/>
    < property name=”returnPageInfo” value=”none”/>
< /plugin >
3.具体使用
在查询之前进行分页： PageHelper.startPage(1,10,true); //第一页 每页显示10条
