当我们在使用分库分表或者一致性缓存的问题，一定会考虑到如何将数据均匀的分散到各个节点之中，以减少数据节点的加减对数据迁移的影响，而一致性Hash算法就是来解决这个问题的。

一致性Hash算法（概念）

一致性Hash算法是将所有的节点通过Hash函数算出来的Hash值散列到环上面，其范围在 0~2^32-1。

之后将节点散列到环上面，可以根据节点的ip或者其他的唯一表示字段作为key通过Hash（Key）算出Hash值散列到环上面，这里的Hash函数可以在网上搜索具体的比较好的实现，我采用的是FNV1_32_HASH算法具体见下文代码

然后如果数据过来的话,也把数据根据Hash函数散列到环上面，这样的话数据k1就到N1节点，数据k2就到N2节点，数据k3就到N3节点

容错性

当N1节点宕机的时候,影响的只是k1数据，k1被重新映射到了k3节点，k2，k3数据并没有发生迁移，这也就保证了容错性

扩展性

当增加一个节点N4的时候，受影响的只是数据K3，这也就保证了环状节点的扩展性

虚拟几点

一致性Hash算法也有他自己的问题，因为Hash算法不可能保障真正的节点的均匀分布，所以会存在节点分布不均匀的情况所以这个时候就需要引入虚拟节点，用虚拟节点形成二次映射，可以很大程度的解决这个问题。具体的话虚拟节点可以用映射节点+编号来生成Hash函数具体见下文。

一致性Hash算法（实现）

一。自定义有序的map

1.初始化一个长度为 N 的数组。
2.将服务节点通过 hash 算法得到的正整数，同时将节点自身的数据（hashcode、ip、端口等）存放在这里。
3.完成节点存放后将整个数组进行排序（排序算法有多种）。
4.客户端获取路由节点时，将自身进行 hash 也得到一个正整数；
遍历这个数组直到找到一个数据大于等于当前客户端的 hash 值，就将当前节点作为该客户端所路由的节点。
5.如果没有发现比客户端大的数据就返回第一个节点（满足环的特性）。
